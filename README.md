# mk-portal-archive

## 项目简介

`mk-portal-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的集中存储与分发点。适用于需要长期保存或分享的 HTML 资源文件。

## 目录结构

```
mk-portal-archive/
├── pages/           # 存放归档的 HTML 页面文件
├── assets/          # 页面引用的公共资源（如图片、样式、脚本）
├── index.html       # 可选：仓库主页导航
└── README.md        # 本文件
```

- **pages/**：每个页面以独立文件形式存放，文件名建议反映内容或日期。
- **assets/**：存放页面间共享的静态资源，避免重复引用。

## 页面归档说明

本仓库内的 HTML 页面均为独立归档，不依赖外部后端服务。每个页面可单独打开运行，内容不包含任何商业推广或外部链接跳转。归档页面可能包含以下类型：

- 信息展示页
- 工具或功能演示页
- 静态内容记录页

如需添加新页面，请遵循以下约定：

1. 将 HTML 文件放入 `pages/` 目录。
2. 如有独立资源，可一并放入 `assets/` 下的子目录中。
3. 保持文件命名简洁，避免特殊字符。

## 维护说明

本仓库由维护者不定期更新。欢迎通过以下方式参与：

- 提交 Issue 报告页面问题或提出建议
- 提交 Pull Request 添加新的归档页面
- 请勿提交包含恶意代码、广告或侵权内容的页面

维护者会审核所有合并请求，确保内容符合归档用途。
